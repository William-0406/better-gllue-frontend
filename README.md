# 更好的谷露前端

这是一个给招聘顾问用的浏览器插件。

它主要做两件事：

- 在脉脉看候选人时，帮你判断这个人谷露里可能有没有。
- 在导入或跟进前，减少重复录入、重复沟通、重复推荐。

简单说：**你在脉脉看人，插件帮你先看一眼谷露里有没有类似记录。**

## 谁适合用

- 经常在脉脉找候选人的顾问。
- 录入候选人前需要反复回谷露查重的人。
- 团队里担心多人重复跟进同一个候选人的顾问。

如果你只是普通顾问，不需要理解代码，也不需要自己改配置。你只需要拿到已经打包好的插件文件，然后按下面步骤安装。

## 普通顾问怎么安装

先向管理员或技术同事要一个已经打包好的插件文件夹，一般名字叫：

```text
dist-extension
```

或者你会收到一个压缩包，解压后里面会有这个文件夹。

### Edge 浏览器

1. 打开 Edge。
2. 地址栏输入 `edge://extensions/`，按回车。
3. 打开“开发人员模式”。
4. 点击“加载解压缩的扩展”。
5. 选择 `dist-extension` 文件夹。
6. 浏览器右上角出现插件图标，就说明安装好了。

### Chrome 浏览器

1. 打开 Chrome。
2. 地址栏输入 `chrome://extensions/`，按回车。
3. 打开“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择 `dist-extension` 文件夹。
6. 浏览器右上角出现插件图标，就说明安装好了。

## 普通顾问怎么使用

### 1. 先登录谷露

先打开你们公司的谷露 CRM，并正常登录。

插件不会替你登录，也不会保存你的账号密码。它只能读取你当前谷露账号本来就有权限看到的数据。

### 2. 再打开脉脉候选人页面

进入脉脉人才详情页后，插件会尝试根据页面上的候选人信息做查重提示。

你重点看这几类信息：

- 是否提示“可能已在库”。
- 是否有相似候选人线索。
- 是否建议先回谷露查看原记录。

### 3. 根据提示决定下一步

如果提示可能已在库：

- 先去谷露确认原记录。
- 看是否已经有人跟进。
- 不要立刻重复新建候选人。

如果没有明显命中：

- 可以继续正常录入。
- 但遇到姓名、公司、职位很常见的人，仍建议人工再确认一次。

## 主要功能

### 脉脉查重提示

在脉脉人才详情页，插件会尽量读取候选人的姓名、公司、职位等信息，并和谷露中的候选人线索做比对。

它能帮助你：

- 少切页面。
- 少复制粘贴。
- 少重复搜索。
- 更早发现重复候选人。

### 脉脉导入前辅助判断

在准备把脉脉候选人录入谷露前，插件会给你一个辅助判断。

它适合回答这些问题：

- 这个人是不是已经进过库？
- 这个人是不是只是公司或职位写法不一样？
- 有没有可能已经被别的顾问跟进？
- 我是不是应该先看谷露原记录？

### 谷露页面增强

插件也提供谷露页面里的只读增强视图，方便快速浏览人才、公司、项目和本周业务进展。

注意：新增、编辑、备注、推荐等操作仍然回到谷露原页面完成。

## 使用后能带来什么效果

- 查重动作更靠前：看脉脉时就能发现风险。
- 录入更谨慎：减少重复建人。
- 团队协作更顺：减少撞候选人、撞沟通。
- 工作流更短：不用每看一个人都手动回谷露搜索。

它不是替你做决定，而是帮你更快发现“这个人可能要先查一下”。

## 当前短板

- 页面信息太少时，查重不一定准。
- 姓名、公司、职位写法不一致时，可能漏判。
- 同名候选人较多时，可能出现误判，需要人工确认。
- 脉脉或谷露页面改版后，插件可能需要更新。
- 如果没有部署增强服务，部分更深的查重或 OCR 能力不可用。

## 使用风险

- 插件提示只能作为辅助，不能当作最终结论。
- 谷露账号权限不足时，插件也看不到更多数据。
- 不要把候选人截图、内部地址、日志、配置文件发到公开地方。
- 如果插件异常、页面没有提示、提示明显不对，请回到谷露原页面人工确认。

## 给管理员或技术同事

普通顾问不需要看这一段。只有需要自己打包插件的人，才需要下面步骤。

### 下载源码

```bash
git clone https://github.com/William-0406/better-gllue-frontend.git
cd better-gllue-frontend
```

### 配置地址

复制 `.env.example` 为 `.env.local`，填写你们自己的谷露地址：

```env
VITE_GLLUE_HOST=your-gllue-host.example.com
VITE_ENHANCE_HOST=
```

说明：

- `VITE_GLLUE_HOST`：谷露 CRM 主机名，不要带 `http://` 或 `https://`。
- `VITE_ENHANCE_HOST`：可选增强服务地址；没有部署增强服务就留空。
- `.env.local` 不要提交到 GitHub。

### 构建插件

```bash
npm install
npm run build:extension
```

构建完成后，把生成的 `dist-extension` 文件夹发给顾问安装。
